06 AI & Prompting 12 skills
#061
Role + context + constraint framing
Every strong prompt has three parts: who Claude is, what it knows, and what it can't do. Master this and outputs jump a full tier.
#062
Chain-of-thought forcing
Add "think step by step before answering" to any analytical prompt. Reduces hallucinations and surfaces reasoning you can audit.
#063
Few-shot example loading
Give Claude 2–3 examples of the exact output format you want before asking for the real thing. Consistency improves 80%.
#064
Negative constraint prompting
"Do NOT use bullet points. Do NOT be vague. Do NOT summarize." Negatives often work better than positives for format control.
#065
Persona locking
Define Claude's persona at the top of every session and it holds character throughout — useful for long projects and consistent voice.
#066
Output format specification
Tell Claude exactly how to structure the response — JSON, table, numbered list, paragraph — before it writes a word.
#067
Iterative refinement loop
First pass = rough output. Then: "Now make it 30% shorter." Then: "Add one concrete example." Each pass sharpens the knife.
#068
Multi-perspective forcing
"Give me 3 takes on this: an optimist, a skeptic, and a practitioner." Breaks Claude out of single-answer mode.
#069
Prompt chaining
Break complex tasks into sequential prompts where each output feeds the next. More reliable than one massive prompt.
#070
Hallucination checking
After any factual response, ask: "Which claims in your answer are you least confident about?" Claude self-flags the weak spots.
#071
Temperature simulation via prompt
"Be creative and surprising" vs "Be precise and conservative" changes output behavior without touching API settings.
#072
System prompt design
Build a reusable system prompt for recurring tasks. Invest 20 minutes once — save hours every week. Your instructions, persistent.
